# 项目设计适配表 - Framebench / 拉片工作台

## 项目

```text
项目名：Framebench / 拉片工作台
项目类型：视频内容分析与报告生成
主要用户：剪辑师、导演、内容创作者
主要场景：上传视频进行 AI 镜头分析，查看镜头维度报告。
```

## 一句话

```text
这个项目帮助用户：通过 AI 深度拆解视频镜头，生成专业的拉片报告。
```

## 工作单

```text
这个项目最核心的工作单是：视频分析任务
工作单标题通常是：视频文件名
一句结论通常回答：分析进度（如 "已发现 42 个镜头，正在分析画面维度"）
```

## 状态豆

只保留必要状态。

```text
检查中：分析中 (Analyzing)
可继续：需确认 (Needs Confirmation)
已完成：已分析 (Completed)
有问题：已中断 (Failed)
可重试：重试 (Retry)
等待中：排队中 (Pending)
```

## 证据条

```text
证据类型 1：镜头关键帧 (Keyframe)
证据类型 2：镜头参数 (Shot Details: duration, type, etc.)
证据类型 3：分析日志 (Thinking/Processing logs)
证据类型 4：拉片报告 (Markdown Report)
```

## 主动作

每个主要场景只允许一个最突出的动作。

```text
默认主动作：开始分析 (Start)
生成类主动作：生成报告 (Generate Report)
修复类主动作：重新分析 (Retry)
确认类主动作：确认并导出 (Confirm & Export)
```

## 项目专属强调

母系统不变，只允许少量项目适配。

```text
项目强调色：#7A553C (Primary Coffee)
项目专属图形：镜头框 (Shot frame)
项目禁用风格：冷蓝色科技感、过多阴影
```

## 文案样例

```text
空状态：拖入视频，我帮你分析
加载中：正在整理镜头...
完成：报告已准备好
失败：分析遇到问题
需要确认：发现了一些模糊镜头，需要你确认
```

## 检查

- 是否只有一个主动作？ 是 (开始分析 / 查看报告)
- 是否用了统一状态豆？ 是
- 是否把复杂内容放进证据条？ 是 (镜头列表)
- 是否少文字但能理解？ 是
- 是否仍然属于暖米咖 AI 工作台体系？ 是
